Error Handling

JSON Error Messages

The Programmable Network API uses an extended error message format with unique error codes, a descriptive message and an auxiliary message generally describing the error in more detail. The schema of an error response is:

{ 
    "type": "object", 
    "properties": { 
        "error-code": { 
            "required": true, 
            "type": "integer" 
        }, 
        "error-message": { 
            "required": true, 
            "type": "string" 
        }, 
        "error-auxiliary": { 
            "required": true, 
            "type": "string" 
        } 
    } 
}

An example:

{ 
  "error-code": 10001, 
  "error-message": "This operation is not supported.", 
  "error-auxiliary": "Error" 
}
